Detailed Description

The runtime client module provides the tools necessary to connect to and interact with the Ilium VR runtime service. This is generally where a beginner to this API will start their work. An example of very basic usage would be the following:

#include <string.h>
#include <ivr.h>
#include <ivr/runtimeclient.h>
ivr_runtime_client* client;
void init()
{
ivr_runtime_client_params params;
params.mode = IVR_RUNTIME_MODE_GAME;
params.options = 0;
params.timeout = 2000;
//These two sleep calls are currently necessary.
ivrRuntimeQueryHidDevices(&params.dev_name);
Sleep(100);
ivrRuntimeQuerySerialDevices(&params.sdev_name);
Sleep(100);
//Just use the first device no matter what
char* newline = strchr(params.dev_name, '\n');
if (newline) newline = '\0';
newline = strchr(params.sdev_name, '\n');
if (newline) newline = '\0';
client = ivrRuntimeClientInit(&client, &params);
ivrRuntimeQueryFreeStr(&params.dev_name);
ivrRuntimeQueryFreeStr(&params.sdev_name);
}
void update(my_game_object* obj)
{
ivr_game_packet packet;
ivrRuntimeReadGame(client, &packet);
obj->rotation = packet.imu;
if (packet.trig < 100)
myGameFireWeapon(obj);
}

This example omits error checking for simplicity and readability.

Data Fields
char * dev_name The device name as returned by ivrRuntimeQueryHidDevices.
See also
ivrRuntimeQueryHidDevices
char * sdev_name The camera device name as returned by ivrRuntimeQuerySerialDevices.
See also
ivrRuntimeQuerySerialDevices
char * mode The way the runtime will operate and provide data. Must be one of the following values:
  • IVR_RUNTIME_MODE_RAW
  • IVR_RUNTIME_MODE_GAME
  • IVR_RUNTIME_MODE_CALIBRATION
  • IVR_RUNTIME_MODE_CAPTURE
DWORD options A list of options to start a runtime with. Only for internal use currently.
DWORD timeout The amount of time, in milliseconds, to wait for any action from the client before automatically terminating the runtime.

A value of 0 is an infinite timeout.

struct ivr_runtime_client

Contains a connection to a runtime and allows a client to interact with it.

This struct is best treated as an opaque pointer as the internals are likely to change at this very early stage, especially when more work is put into making the runtime work cross-platform

struct ivr_raw_packet

A packet returned by ivrRuntimeClientReadRaw when the runtime is set to the IVR_RUNTIME_MODE_RAW mode.

Data Fields
ivr_vec3 a Calibrated accelerometer data.
ivr_vec3 g Calibrated gyroscope data.
uint8_t joybtn Controller button states. Supports up to 8 buttons.
uint8_t trig Controller trigger state. A value of 0 is fully pressed, 255 is fully released.
uint8_t joyx Controller joystick X component.
uint8_t joyy Controller joystick Y component.
uint32_t numBlobs The number of blobs found by the camera.
See also
ivr_blob
ivr_blob * blobs A pointer to an array of camera blobs. Null if numBlobs == 0.
struct ivr_game_packet

A packet returned by ivrRuntimeClientReadGame when the runtime is set to the IVR_RUNTIME_MODE_GAME mode.

Data Fields
ivr_vec3 imu Fused sensor data representing orientation as Euler angles in degrees.
uint8_t joybtn Controller button states. Supports up to 8 buttons.
uint8_t trig Controller trigger state. A value of 0 is fully pressed, 255 is fully released.
uint8_t joyx Controller joystick X component.
uint8_t joyy Controller joystick Y component.
ivr_quat camRot An estimated orientation of the device based on camera data.
ivr_vec3 camPos An estimated position of the device based on camera data.

ivr_error_code ivrRuntimeQueryHidDevices ( char **  devices)

Queries the runtime service for attached Ilium VR devices. Names are delimited by newline.

Parameters
[out]devicesA pointer to a string that will be allocated over and filled with Ilium VR devices names from the runtime.
Returns
If the function is successful, it will return IVR_E_OK. Otherwise, devices will be set to NULL and this function will return one of the following errors:
Value Description
IVR_E_OUT_OF_MEM No available system memory
IVR_E_ARG_NULL devices is NULL
GetLastError() Windows error code
